Documentação v0.9.0

Como a regra funciona por trás da interface.

Um resumo técnico da lógica de escalas, da organização em camadas e da persistência com JSON e PostgreSQL.

Ciclo por dias

Usa a soma dos dias trabalhados e dias de folga para formar um ciclo.

ciclo = dias_trabalho + dias_folga
posicao = dias_passados % ciclo
status = "Trabalhando" se posicao < dias_trabalho

Ciclo por horas

Segue o mesmo princípio, mas trabalha com data e hora. É útil para escalas como 12x36.

ciclo = horas_trabalho + horas_folga
posicao = horas_passadas % ciclo
status = "Trabalhando" se posicao < horas_trabalho

Turno rotativo

Usa uma sequência personalizada. Cada posição da lista representa um dia do ciclo.

sequencia = ["Manhã", "Tarde", "Noite", "Folga"]
posicao = dias_passados % len(sequencia)
status = sequencia[posicao]

Persistência

Na v0.9.0, a aplicação pode usar JSON ou PostgreSQL. A escolha é técnica e feita por variável de ambiente. 

ESCALA_REPOSITORY=json      # modo padrão
ESCALA_REPOSITORY=postgres  # usa PostgreSQL

POSTGRES_USER=postgres
POSTGRES_PASSWORD=SUA_SENHA_AQUI
POSTGRES_DB=simulador_escala
POSTGRES_HOST=127.0.0.1
POSTGRES_PORT=5432

Schema PostgreSQL

O banco foi dividido por tipo de escala.

A tabela principal guarda nome e tipo. As tabelas específicas guardam os campos de cada modelo.

escalas

Tabela principal com id, nome, tipo, criado_em e atualizado_em.

escalas_ciclo_dias

Guarda dias_trabalho e dias_folga.

escalas_ciclo_horas

Guarda horas_trabalho e horas_folga.

escalas_turnos

Guarda a sequência ordenada de turnos rotativos.
