Por que esse projeto existe?

A dor é simples: escala confusa vira erro, atraso e insegurança.

Quem trabalha em escala nem sempre consegue bater o olho no calendário e saber rapidamente se estará trabalhando, de folga ou em qual turno estará. Para empresas, essa falta de clareza também dificulta planejamento, comunicação e organização de equipes.

Para o colaborador

Evita dúvidas sobre folgas, plantões, trocas e dias futuros.

Para a empresa

Ajuda a visualizar padrões, reduzir confusão operacional e padronizar escalas.

Para o portfólio

Mostra Python aplicado em um problema real, com testes, JSON, CLI e interface web.

Documentação

Entenda como o projeto está organizado.

Ciclo por dias

Usa uma data inicial, dias trabalhados e dias de folga. A posição no ciclo define o status de cada dia. 

ciclo = dias_trabalho + dias_folga
posicao = dias_passados % ciclo

Ciclo por horas

Ideal para escalas como 12x36. O cálculo usa data e hora inicial, somando períodos de trabalho e folga. 

ciclo_horas = horas_trabalho + horas_folga
posicao = horas_passadas % ciclo_horas

Turno rotativo

A v0.6 começa a preparar sequências como manhã, tarde, noite e folga. A cada dia, o simulador avança uma posição. 

["Manhã", "Tarde", "Noite", "Folga"]

Sobre o projeto

Um projeto de estudo com cara de solução real.

O simulador nasceu para praticar Python com um problema próximo da rotina de trabalho: entender escalas, prever folgas e organizar informações em uma aplicação simples.

Stack usada

  • Python para lógica principal
  • JSON para persistência local
  • Pytest para testes automatizados
  • HTML, CSS e JavaScript para demo web
  • GitHub para versionamento e portfólio

Limites importantes

Esta página é uma demonstração educacional. Os dados ficam no navegador do usuário por meio de localStorage. O simulador não deve ser usado como sistema oficial de controle de ponto, folha de pagamento ou gestão trabalhista sem validação profissional e jurídica.
