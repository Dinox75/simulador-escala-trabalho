Documentação

Como a lógica funciona

A demo resume a lógica principal do projeto Python e mostra como o cálculo evolui entre ciclos por dias e ciclos por horas.

Lógica de ciclo por dias

Soma os dias trabalhados com os dias de folga, encontra a posição da data dentro do ciclo e retorna se o dia é de trabalho ou folga.

ciclo = dias_trabalho + dias_folga
posicao = dias_passados % ciclo

se posicao < dias_trabalho:
    Trabalhando
senão:
    Folga

Lógica de ciclo por horas

Para escalas como 12x36, o cálculo considera data e hora. A visualização deixa de ser apenas por dia e passa a ser por períodos.

ciclo_horas = horas_trabalho + horas_folga
posicao = horas_passadas % ciclo_horas

se posicao < horas_trabalho:
    Trabalhando
senão:
    Folga

Arquitetura atual

  • main.py: controla o fluxo principal do terminal.
  • escala.py: concentra os cálculos de escala.
  • armazenamento.py: manipula o JSON de escalas salvas.
  • validacoes.py: valida entradas do usuário.
  • interface.py: concentra mensagens e exibições da CLI.
  • docs/demo: apresenta a versão visual estática.

Limites atuais

Esta demo é estática e usa localStorage. Ela é excelente para apresentação e validação visual, mas ainda não substitui um sistema real de RH.

Importante: férias, feriados, autenticação, banco de dados, auditoria e regras legais precisam ser tratados antes de qualquer uso corporativo real.

Termos de uso

Uso da demonstração

Estes termos são simples e servem para contextualizar a demo como projeto educacional e de portfólio.

1. Finalidade

Esta página é uma demonstração educacional e visual do projeto Simulador de Escala de Trabalho. Ela não substitui sistemas oficiais de RH, folha de pagamento ou controle de jornada.

2. Dados armazenados

As escalas criadas nesta demo são salvas apenas no navegador do usuário usando localStorage. Nenhum dado é enviado para servidor externo por esta página estática.

3. Precisão das informações

A simulação depende das informações inseridas. Se a data inicial ou os valores de escala estiverem incorretos, o resultado também poderá estar incorreto.

4. Uso profissional

Para uso corporativo real, o projeto precisaria evoluir com autenticação, banco de dados, permissões, auditoria, logs, validações legais e regras internas.
