Por que o projeto existe?

A dor é simples: escala mal explicada vira confusão.

Quando a escala fica em grupo, papel, planilha solta ou memória de alguém, a dúvida aparece: hoje eu trabalho? qual turno? quando folgo? O projeto nasceu para transformar essa regra em consulta visual.

A proposta é validar uma ferramenta objetiva, com boa usabilidade e base técnica evolutiva. Primeiro terminal, depois demo web, agora banco de dados e arquitetura mais profissional.

O que mudou na v0.9.0

O projeto saiu do JSON isolado e ganhou persistência real com PostgreSQL.

A aplicação continua simples para rodar em JSON, mas agora também possui banco relacional, repository dedicado, schema SQL e testes de integração.

PostgreSQL funcional

Cadastro, listagem, busca e exclusão de escalas funcionando em banco relacional.

Repository Pattern

O projeto separa regra de negócio da persistência usando repositories para JSON e PostgreSQL.

Configuração segura

Senha e dados do banco ficam fora do código, usando variáveis de ambiente.

Testes automatizados

O fluxo antigo foi preservado e os novos recursos foram protegidos com pytest.
