Por que o projeto existe?

A dor é simples: escala mal explicada vira confusão.

Muitos colaboradores precisam perguntar se trabalham, quando folgam ou qual turno está valendo. Empresas também sofrem quando a regra está espalhada em grupos, planilhas ou anotações soltas.

Dúvida recorrente

O colaborador perde tempo confirmando algo que deveria estar claro em segundos.

Folgas difíceis de prever

Escalas como 6x3, 12x36 e turnos rotativos não são tão intuitivas quando vistas só em texto.

Gestão sem visão rápida

Para a empresa, visualizar padrões ajuda a evitar ruído, erro operacional e retrabalho.

Visão de futuro

Duas áreas futuras para transformar a demo em produto.

A ideia é evoluir de um simulador de portfólio para uma ferramenta com valor real para pessoas e empresas.

Área do colaborador

Um espaço simples para consultar hoje, próximos dias, folgas, troca de turno e alertas personalizados.

  • Consulta rápida da escala atual
  • Calendário mensal com folgas destacadas
  • Notificações antes de mudanças de turno
  • Histórico de escalas aplicadas

Área da empresa

Um painel para cadastrar equipes, padrões de escala e acompanhar a distribuição de turnos.

  • Cadastro de setores e equipes
  • Gestão de modelos de escala
  • Visão macro de cobertura operacional
  • Exportação de relatórios
Documentação

Como a lógica funciona por trás da interface.

Ciclo por dias

Calcula a posição do dia consultado dentro de um ciclo formado por dias trabalhados + dias de folga.

ciclo = dias_trabalho + dias_folga
posição = dias_passados % ciclo

Ciclo por horas

Usado para escalas como 12x36, onde a regra depende de data e hora inicial.

ciclo = horas_trabalho + horas_folga
posição = horas_passadas % ciclo

Turno rotativo

Usa uma sequência definida pelo usuário, como Manhã → Tarde → Noite → Folga, e retorna a etapa correspondente ao dia consultado.

sequência = ["Manhã", "Tarde", "Noite", "Folga"]
turno = sequência[dias_passados % len(sequência)]

Sobre o projeto

Um projeto Python que cresceu para uma experiência visual.

O Simulador de Escala de Trabalho começou como um projeto de lógica em Python para consultar dias de trabalho e folga. A demo web transforma essa lógica em uma experiência mais amigável, com visualização por cards, linha do tempo e calendário.

Termos de uso

Esta página é uma demonstração educacional e de portfólio. Ela não substitui sistemas oficiais de RH, controle de jornada ou gestão trabalhista.

Dados locais

As escalas criadas na demo ficam salvas apenas no navegador por meio de localStorage. Nenhuma informação é enviada para servidor.

Limitações

Feriados, banco de horas, trocas de turno e regras legais específicas ainda não fazem parte da simulação.
