Por que o projeto existe?

A dor é simples: escala mal explicada vira confusão.

Quando a escala fica em conversa de grupo, papel, planilha solta ou memória de alguém, a dúvida aparece sempre: hoje eu trabalho? qual turno? quando folgo? O projeto nasceu para transformar essa regra em consulta visual.

A proposta não é substituir um sistema corporativo completo agora. A ideia inicial é validar uma ferramenta objetiva, com boa usabilidade, que ajude a reduzir ruído entre colaborador, liderança e rotina real de operação.

Documentação

Como a regra funciona por trás da interface.

Um resumo técnico para quem quer entender a lógica do projeto antes de olhar o código.

Ciclo por dias

Usa a soma dos dias trabalhados e dias de folga para formar um ciclo. Depois calcula a posição da data consultada dentro desse ciclo.

ciclo = dias_trabalho + dias_folga
posicao = dias_passados % ciclo
status = "Trabalhando" se posicao < dias_trabalho

Ciclo por horas

Segue o mesmo princípio, mas usa horas. É útil para escalas como 12x36, em que a virada pode acontecer no mesmo dia ou atravessar madrugada.

ciclo = horas_trabalho + horas_folga
posicao = horas_passadas % ciclo
status = "Trabalhando" se posicao < horas_trabalho

Turno rotativo

Usa uma sequência personalizada. Cada posição da lista representa um dia do ciclo. Quando chega ao fim, a sequência recomeça.

sequencia = ["Manhã", "Tarde", "Noite", "Folga"]
posicao = dias_passados % len(sequencia)
status = sequencia[posicao]

Modelos predefinidos

Na v0.7.0, os modelos ficam centralizados no arquivo modelos_escala.py. A demo web replica essa ideia usando uma lista de modelos no JavaScript.

modelos = [

Arquitetura inicial

O projeto Python separa responsabilidades: cálculo em escala.py, dados em armazenamento.py, modelos em modelos_escala.py, validações em validacoes.py e exibição em interface.py.

main.py → fluxo do programa
escala.py → regras de cálculo
armazenamento.py → JSON
interface.py → saída visual
validacoes.py → entradas seguras
modelos_escala.py → modelos prontos
Sobre o projeto

Um projeto de portfólio com problema real, evolução incremental e foco em usabilidade.

O Simulador de Escala de Trabalho nasceu como um projeto Python de linha de comando para praticar fundamentos importantes: datas, funções, modularização, persistência em JSON, validação de entrada e testes automatizados.

A evolução natural foi transformar a lógica em uma demo web. Assim, o projeto deixa de ser apenas um script técnico e passa a comunicar valor para usuários reais: quem trabalha em escala e quem precisa organizar equipes.

A proposta atual é manter o núcleo simples e confiável, enquanto a interface ganha clareza visual. Cada versão adiciona uma camada: primeiro ciclos por dias, depois ciclos por horas, depois turnos rotativos e agora modelos predefinidos com uma escala real de 24 dias.

Termos de uso

Uso responsável da demo.

Esta página apresenta uma simulação para fins educacionais, técnicos e demonstrativos.

1. Finalidade

O simulador é um projeto de estudo e portfólio. Ele demonstra cálculos de escala com base em dados informados pelo usuário, sem substituir sistemas oficiais de RH, folha, ponto ou gestão corporativa.

2. Conferência obrigatória

Antes de usar qualquer resultado em contexto real, confira com a empresa, liderança ou documento oficial da escala. A demo ajuda na visualização, mas não deve ser a única fonte de decisão.

3. Dados locais

As escalas salvas na demonstração ficam no navegador por meio de localStorage. Elas não são enviadas para servidor e podem ser apagadas ao limpar os dados do navegador.

4. Limitações

Regras especiais como feriados, banco de horas, trocas de turno, afastamentos, férias, faltas, horas extras e exceções operacionais ainda não fazem parte do cálculo padrão.

5. Evolução do projeto

Novas versões podem alterar textos, interface, estrutura de dados e comportamento da demo. O objetivo é evoluir com testes e melhorias graduais.

6. Responsabilidade

O uso da ferramenta é livre para estudo e avaliação. Em caso de uso profissional, recomenda-se validação técnica, jurídica e operacional antes de qualquer implantação real.
