Documentação

Como o projeto funciona

Esta área resume a lógica do sistema, a evolução da v0.5.0 e o que ainda será implementado.

Lógica de ciclo por dias

Soma os dias trabalhados com os dias de folga, encontra a posição da data consultada dentro do ciclo e define o status. 

ciclo = dias_trabalho + dias_folga
posicao_ciclo = dias_passados % ciclo

se posicao_ciclo < dias_trabalho:
    Trabalhando
senão:
    Folga

Lógica de ciclo por horas

Para escalas como 12x36, a lógica precisa considerar data e hora. O ciclo é calculado em horas e a visualização é feita por períodos com início e fim. 

ciclo_horas = horas_trabalho + horas_folga
horas_passadas = data_hora_consulta - data_hora_inicio
posicao_ciclo = horas_passadas % ciclo_horas

se posicao_ciclo < horas_trabalho:
    Trabalhando
senão:
    Folga

Arquitetura atual

A versão Python foi separada em módulos para facilitar manutenção, testes e evolução.

  • main.py: controla o fluxo principal do terminal.
  • escala.py: concentra o cálculo das escalas.
  • armazenamento.py: manipula o JSON de escalas salvas.
  • tipos_escala.py: centraliza os tipos suportados.
  • validacoes.py: valida entradas do usuário.
  • interface.py: organiza as mensagens exibidas no terminal.

Sobre

Um projeto de estudo com visão de produto real

Por que existe?

O projeto nasceu para resolver uma dor prática: consultar rapidamente se uma data ou horário futuro será de trabalho ou folga em uma escala cíclica.

Por que é bom para portfólio?

Ele mostra fundamentos reais: modularização, validação, persistência, testes, documentação, versionamento e evolução incremental.

Visão futura

O sistema pode evoluir para uma plataforma de gestão de escalas com colaboradores, setores, turnos, férias, folgas e relatórios.

Termos de uso

Uso da demonstração

Estes termos são um modelo simples para apresentação do projeto e não substituem revisão jurídica.

1. Finalidade

Esta página é uma demonstração educacional e visual do projeto Simulador de Escala de Trabalho. Ela não substitui sistemas oficiais de RH, folha de pagamento ou controle de jornada.

2. Dados armazenados

As escalas criadas nesta demo são salvas apenas no navegador do usuário usando localStorage. Nenhum dado é enviado para servidor externo por esta página estática.

3. Precisão das informações

A simulação é baseada nas informações inseridas pelo usuário. Caso a data inicial ou a configuração da escala esteja incorreta, o resultado também poderá ficar incorreto.

4. Uso profissional

Para uso corporativo real, o projeto precisaria evoluir com autenticação, banco de dados, controle de permissões, auditoria, tratamento de exceções e validações legais aplicáveis.

5. Responsabilidade

Esta demo tem finalidade de estudo, portfólio e apresentação técnica. A responsabilidade pelo uso das informações simuladas é do usuário.
